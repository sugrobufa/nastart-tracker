# 🏊 НАСТАРТ Трекер

Трекер прогресса юных пловцов по результатам соревнований [НАСТАРТ](https://настарт.рф).

**3254 спортсмена · 12186 результатов · 15 соревнований (2022–2026)**

## Возможности

- Поиск спортсмена по фамилии
- Лидерборд по возрастным группам с переключением короткой/длинной воды
- **Рейтинг школ плавания** — по баллам и эффективности (баллы ÷ количество пловцов)
- **Рейтинг тренеров** — по баллам и эффективности с подсчётом медалей
- **Методология рейтинга** — информационная кнопка с описанием системы начисления баллов
- График динамики результатов с переключателем бассейна (25м / 50м)
- Аллея славы — медали за места в тройке призёров
- Рейтинг и прогресс спортсменов
- История клубов и тренеров по годам
- Таблица результатов с фильтрами по дисциплине, соревнованию и клубу
- Разряды ЕВСК — автоматическое определение (III юн, II юн, I юн, III, II, I, КМС)
- Отметка (к) для результатов в 25м бассейне
- Кнопка «Назад к поиску» из карточки спортсмена
- Тёмная и светлая тема
- Адаптивная мобильная версия
- **Автообновление** — еженедельная проверка новых протоколов

## Быстрый старт

Сайт — статический HTML/CSS/JS. Для просмотра просто откройте `index.html` в браузере.

### Публикация на GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите все файлы из этого архива (сохраняя структуру папок)
3. Перейдите в **Settings → Pages → Source → GitHub Actions**
4. Сайт будет доступен по адресу `https://ваш-логин.github.io/имя-репозитория/`

### Привязка своего домена

1. В **Settings → Pages → Custom domain** укажите свой домен
2. У регистратора домена добавьте CNAME запись → `ваш-логин.github.io`
3. Включите **Enforce HTTPS**

## Автообновление (полная автоматика)

После загрузки репозитория на GitHub, данные обновляются **полностью автоматически**:

### Как это работает

1. **Каждый понедельник в 10:00 МСК** GitHub Actions запускает `auto_check.py`
2. Скрипт проверяет два источника:
   - [настарт.рф](https://настарт.рф) — раздел «Прошедшие старты» с итоговыми протоколами
   - [Telegram-канал](https://t.me/s/swimschool1cup) — посты со ссылками на протоколы
3. Если найдены **новые** протоколы (которых ещё нет в базе):
   - Скачивает PDF с Яндекс.Диска
   - Определяет метаданные (дата, тур, возрастная группа)
   - Обновляет список PROTOCOLS в parse_pdfs.py
   - Запускает парсинг и дедупликацию
   - Коммитит изменения → автоматически деплоится на GitHub Pages
4. Если новых протоколов нет — ничего не происходит

### Ручной запуск

Можно запустить проверку в любой момент:
**Actions → Автообновление данных → Run workflow**

### Настройка расписания

Расписание задаётся в `.github/workflows/update-data.yml`:
```yaml
schedule:
  - cron: '0 7 * * 1'  # Понедельник 07:00 UTC = 10:00 МСК
```

Другие варианты:
- `'0 7 * * 1,4'` — понедельник и четверг
- `'0 7 * * *'` — каждый день
- `'0 */6 * * *'` — каждые 6 часов

## Обновление данных вручную

Если автоматика не нашла протокол, можно добавить его вручную:

1. **Скачайте PDF протокол** с [настарт.рф](https://настарт.рф) или из
   [Telegram-канала](https://t.me/s/swimschool1cup) (ссылки на Яндекс.Диск)

2. **Назовите файл** по формату: `YYYY_MM_DD_Ntour_agegroup.pdf`
   - Пример: `2026_05_17_2tour_7.pdf`, `2026_05_17_2tour_8plus.pdf`

3. **Добавьте запись** в `scripts/parse_pdfs.py` → массив `PROTOCOLS`:
   ```python
   {"name": "2026_05_17_2tour_7", "date": "2026-05-17", "meet": "НАСТАРТ II тур 2026"},
   ```
   - `name` — имя PDF файла без расширения
   - `date` — дата соревнования (YYYY-MM-DD)
   - `meet` — название (формат: `НАСТАРТ N тур YYYY`)
   - Длину бассейна (25м/50м) указывать НЕ НУЖНО — определяется автоматически из PDF

4. **Положите PDF** в папку `pdf_protocols/`

5. **Сделайте git push** — workflow автоматически пересоберёт `data.json`

### Локальный запуск
```bash
pip install -r scripts/requirements.txt
# Автопроверка новых протоколов
python3 scripts/auto_check.py
# Или ручная пересборка
./scripts/update_data.sh
```

## Как скачать PDF с Яндекс.Диска

Ссылки на протоколы обычно ведут на Яндекс.Диск. Для скачивания через командную строку:

```bash
# Получить прямую ссылку
curl -s "https://cloud-api.yandex.net/v1/disk/public/resources/download?public_key=ССЫЛКА" | python3 -c "import sys,json; print(json.load(sys.stdin)['href'])"

# Скачать
curl -L -o pdf_protocols/имя_файла.pdf "ПРЯМАЯ_ССЫЛКА"
```

## Методология рейтинга школ и тренеров

| Место | Баллы |
|-------|--------|
| 1 | 5 |
| 2 | 3 |
| 3 | 2 |
| Личный рекорд (бонус) | +1 |

- **Баллы** = сумма за все старты
- **Эффективность** = баллы ÷ количество активных пловцов / учеников
- Тренеры определяются по формату записи клуба: `Название Школы Фамилия И.О.`

## Технические детали

- **Парсер** поддерживает два метода извлечения данных из PDF: pdfplumber (основной) и PyMuPDF (fallback для PDF с проблемами кодировки)
- **Длина бассейна** определяется автоматически из заголовков PDF (`Бассейн 25 м` / `Бассейн 50 м`)
- **Дедупликация** объединяет записи с/без отчества (например, «Кашапова Анастасия» и «Кашапова Анастасия Павловна»)
- **Нормализация клубов** объединяет варианты написания одного клуба
- **Разряды ЕВСК** определяются по нормативам с учётом пола, дисциплины и длины бассейна
- **auto_check.py** использует только стандартную библиотеку Python (urllib) — без дополнительных зависимостей для HTTP-запросов
- **Возрастные группы**: 2015 и младше, 2013-2014, 2011-2012, 2009-2010, 2010 и старше
